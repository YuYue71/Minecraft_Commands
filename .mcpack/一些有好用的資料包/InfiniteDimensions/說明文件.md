# InfiniteDimensions - Minecraft 無限維度生成模板

本模板專為 Minecraft 1.20.4+ 版本設計, 採用原版 Jigsaw (拼圖) 系統與 Structure Set (區塊平鋪) 技術, 能夠生成無限延伸, 絕對規整且無縫銜接的幾何迷宮維度. 此架構徹底捨棄了傳統的 Noise 噪聲地形生成, 確保維度內只有你親自設計的建築結構, 非常適合用於開發 Backrooms (後室), 巨型地牢, 監獄或無盡迴廊等自訂地圖.

## 核心生成原理

傳統的 Jigsaw 系統(如村莊)具有生成深度限制, 當玩家走遠時會遇到結構邊界. 本模板採用 **Chunk Grid Tiling (區塊網格平鋪法)** 進行駭客級改造:
透過設定 structure_set, 強制遊戲在"每一個 16x16 的區塊"中都必定生成一個房間, 並依靠每個房間邊緣的拼圖方塊互相咬合, 達成效能極佳且永遠走不到盡頭的無限建築世界.

## 資料夾結構概覽

整個資料包的核心運作邏輯分佈於 data/new_dimensions/ 目錄下的各個資料夾中.

| 資料夾路徑 | 系統層級 | 檔案用途說明 |
| --- | --- | --- |
| dimension_type/ | 物理環境層 | 決定維度的光照, 高度限制, 時間流逝與天空特效. |
| dimension/ | 地形畫布層 | 宣告一個沒有任何地形(純空氣)的超平坦畫布供結構生成. |
| structures/ | 模型實體層 | 存放你在遊戲內使用結構方塊匯出的 .nbt 房間檔案. |
| worldgen/template_pool/ | 結構抽卡池 | 定義有哪些房間可以生成, 以及它們出現的權重(機率). |
| worldgen/structure_set/ | 網格佈局層 | 控制結構在世界中分佈的頻率與間距. |
| worldgen/structure/ | 生成設定層 | 指定迷宮生成的絕對高度以及自訂怪物生成規則. |

## 客製化參數指南

你可以透過修改以下 JSON 檔案來完全控制這個無限維度的樣貌. 以下列出所有可修改的關鍵元素:

### 1. 環境與物理規則 (Dimension Type)

**檔案路徑:** data/new_dimensions/dimension_type/example.json

| JSON 鍵值 | 資料型態 | 修改效果說明 |
| --- | --- | --- |
| ambient_light | Float | 控制環境的基礎亮度. 數值範圍 0.0 ~ 1.0. (後室建議設為 0.1 保持昏暗) |
| fixed_time | Integer | 鎖定維度的時間. 例如 6000 為正午, 18000 為午夜. 移除此鍵值則恢復日夜交替. |
| effects | String | 渲染特效. "minecraft:the_end" 可產生終界靜電雜訊感; "minecraft:overworld" 則為一般天空. |
| min_y | Integer | 維度的最低 Y 軸座標. 建議與 height 配合使用. |
| height / logical_height | Integer | 維度的總高度. |
| has_ceiling | Boolean | 若設為 true, 維度頂部會像地獄一樣有基岩層覆蓋感. |

### 2. 迷宮抽卡池與機率 (Template Pool)

**檔案路徑:** data/new_dimensions/worldgen/template_pool/example_pool/rooms.json

這個檔案決定了迷宮是由哪些房間組成的. 可以在 elements 陣列中不斷新增房間.

| JSON 鍵值 | 修改效果說明 |
| --- | --- |
| weight | (重要) 該房間被抽中的權重. 數值越高越容易出現. 例如: 直走廊設為 50, 死胡同設為 5, 藏寶圖大廳設為 1. |
| location | 填寫 .nbt 檔案的命名空間路徑. 例如 "new_dimensions:example_structures/a" 對應 a.nbt. |
| projection | 必須保持為 "rigid" (剛性), 以防止地形起伏導致結構扭曲變形. |

### 3. 網格平鋪密度 (Structure Set)

**檔案路徑:** data/new_dimensions/worldgen/structure_set/example_spawner.json

| JSON 鍵值 | 修改效果說明 |
| --- | --- |
| spacing | 決定結構生成的區塊間隔. 本模板預設為 1, 代表每個區塊(16x16)都必定生成一個房間. 若想改成稀疏的分佈, 可加大此數值. |
| separation | 結構之間的最小距離. 為了達成完美無縫拼接, 必須保持為 0. |

### 4. 生成高度與怪物覆蓋 (Structure)

**檔案路徑:** data/new_dimensions/worldgen/structure/example_maze.json

| JSON 鍵值 | 修改效果說明 |
| --- | --- |
| start_height -> absolute | 決定整個迷宮生成的絕對 Y 軸高度. 請確保此高度落在你的 dimension_type 邊界之內. |
| step | 必須保持為 "raw_generated", 這樣生成器才會無視超平坦沒有地表的問題, 強制置放結構. |
| spawn_overrides | 自訂此迷宮專屬的怪物生成規則. 可在 spawns 陣列中新增怪物種類. 注意: minCount 必須大於 0. |

## 標準開發流程 (SOP)

為了確保你的房間能夠被系統完美拼湊, 請嚴格遵守以下遊戲內建築規範:

1. **規劃 16x16 範圍:**
在遊戲內找一塊空地, 標記出精確的 16x16 區塊邊界. 你的房間主體必須完整包含在這個範圍內.
2. **放置拼圖方塊 (Jigsaw Block):**
在房間四周預計作為出入口的邊緣, 放置拼圖方塊.
* Name 屬性統一命名 (如: maze:door).
* Target Name 屬性需與上述相同 (如: maze:door).
* Target Pool 屬性必須填寫 "minecraft:empty" (因為網格系統已接管生成, 拼圖方塊僅作對齊用).


3. **儲存結構:**
使用結構方塊將房間儲存起來. 務必確認儲存的尺寸剛好是 16xXx16, 並且將檔案匯出成 .nbt 格式.
4. **加入資料包:**
將 .nbt 檔案放入 data/new_dimensions/structures/example_structures/ 中.
5. **登錄抽卡池:**
打開 rooms.json, 複製一段 element 區塊, 將 location 指向你的新房間, 並給予適當的 weight.
6. **進遊戲測試:**
建立全新世界並掛載此資料包, 輸入 /execute in new_dimensions:example run tp @s 0 <你設定的start_height> 0 進行驗收.

## 常見問題排解

**Q1. 進入維度後周遭是一片虛空, 完全沒有方塊?**

* 檢查 1: 確認你的 .nbt 檔案確實放在複數型態的 structures 資料夾中, 而不是 structure.
* 檢查 2: 確認你傳送的高度是否正確. 如果 start_height 設為 -511, 但你傳送到 Y=64 的位置, 你將無法看見深淵底下的結構.
* 檢查 3: 確保 structure 檔案中的 step 設定為 "raw_generated".

**Q2. 載入資料包時顯示驗證失敗 (Validation Failed)?**

* 請開啟遊戲目錄下的 logs/latest.log 查看精確報錯點.
* 常見原因包含 JSON 檔案缺少逗號, 單複數路徑拼寫錯誤, 或是 spawn_overrides 內的 minCount 設為 0 等不合法的原版數值限制.

**Q3. 房間生成了, 但是彼此之間有錯位或產生了不合理的斷層?**

* 這是因為你的結構方塊在儲存 .nbt 時, 沒有精確對齊 Minecraft 預設的區塊網格 (Chunk Borders). 請在遊戲內按下 F3 + G 顯示區塊邊界, 確保你的 16x16 房間精準貼合邊界框後再重新存檔.
