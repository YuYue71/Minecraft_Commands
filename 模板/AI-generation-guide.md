## 生成前核對 Checklist {#checklist}

生成任何文件前，逐項確認後再輸出：

- [ ] 使用正確的模板
- [ ] 所有佔位符已替換，無任何「一句話說明」、`<值>`、`[大類名稱]` 殘留
- [ ] 模板內所有 `> 選填：...` 標注若無內容，整行刪除
- [ ] 「條件依賴」區塊無內容時，整段含標題一起刪除
- [ ] 目錄錨點已對應實際 H2/H3 標題，無錯誤跳轉
- [ ] 枚舉值已查找完整資料並全部列出，無「... 等」省略
- [ ] 有版本限定的欄位或行為已標注 `[1.XX+]`
- [ ] 外部連結 Wiki 項目必填，連結目的地統一填 `#`
- [ ] 最後更新日期填入實際生成日期，格式 `YYYY-MM-DD`
- [ ] 輸出純 Markdown，無額外說明文字包裹

## 標準生成 Prompt 格式 {#prompt-format}

```
請依照 [模板名稱] 模板，生成以下內容的說明文件。

主題：[指令名稱 / 元素名稱 / NBT 大類名稱]
適用版本：[JE 1.XX+]
權限等級：[僅 command-page 需填，其餘省略]
子指令列表：[僅 command-page 需填，列出所有子指令名稱]
版本限定欄位：[列出需標注 [1.XX+] 的欄位或行為，無則填「無」]
條件依賴關係：[描述欄位間相依或互斥規則，無則填「無」]
共用內容引用：[列出應改用連結引用而非重複展開的內容，如目標選擇器]

枚舉資料：由 AI 自行查找，規則見下方「枚舉資料處理規則」

嚴格規則：
1. 模板區塊順序不得更動
2. 所有佔位符必須替換，無殘留
3. 語言：中文主體，技術術語、ID、路徑、欄位名稱保留英文
4. 枚舉值查找完整後全部列出，不省略
5. 輸出純 Markdown，不加任何額外說明文字
```

## 各模板專屬規則 {#template-rules}

### `template-command-page.md`

- **語法區**：每種語法形式獨立一行，只寫最小單元語法，不展開組合
- **子指令區**：每個子指令一個 `###` H3，H3 下方先放該子指令的最小語法 code block，再接欄位說明表
- **無子指令的指令**（如 `/say`、`/time set`）：`子指令說明` 區塊整段替換為單一 `## 參數說明` 區塊，格式與欄位說明表相同
- **欄位說明表各欄定義：**
  - `欄位`：參數原始名稱，保留 `<>` / `[]` 括號
  - `類型`：`entity` / `int` / `string` / `enum` / `boolean` / `block` / `vec3` / `float` 等
  - `可填值 / 範圍`：枚舉值以 `/` 分隔；數值範圍用 `..`；無限制填 `—`
  - `預設`：選填參數的預設值；必填填 `—`
  - `說明`：實際用途描述，不得只寫類型名稱；共用語法改用連結引用
- **超長枚舉（音效、粒子、物品、生物等）**：在對應子指令 H3 下方新增 `#### 枚舉值` H4，加入分類欄，查找完整後照列所有條目，不省略
- **目標選擇器參數**：欄位說明欄只寫「目標選擇器語法，見 [目標選擇器](#)」，不在此展開

---

### `template-reference-doc.md`

- **章節數量不固定**，依內容性質決定，章節標題替換佔位符後同步更新目錄
- **核心結構區**若無明確語法格式，整段替換為條列規則
- **枚舉型表格**加入分類欄；純說明型表格維持兩欄

---

### `template-datapack-illustrate.md`

- **命名空間資料夾表格**連結目的地統一填 `#`
- **`pack_format` 表格**保持完整，不截斷

---

### `template-datapack-element.md`

- **JSON 結構區**：只放精簡完整範本，選填欄位填實際預設值，不羅列所有可能組合
- **欄位說明表**：必填打 `✓`，選填留空，不填 `✗`
- **枚舉值速查區**：只為有明確可填值集合的欄位建表；自由字串欄位（路徑、自訂名稱）不建表
- **子欄位巢狀超過兩層**：每層獨立一個 `###` 子區塊說明，不在同一張表混列
- **目錄中的子欄位條目**：有獨立 `###` 區塊的才列入目錄，不列每一個欄位

---

### `template-nbt-overview.md`

- **大類索引表**連結目的地填 `#`
- **資料類型表與路徑語法表**保持完整，不截斷

---

### `template-nbt-category.md`

- **通用欄位區塊**只列所有子類型共通的 Tag，不列任何子類型專屬欄位
- **子類區塊數量**依實際子分類數量增減，每個子類一個 `##` 區塊，同步更新目錄
- **`compound[]` 欄位**：必須附上 SNBT 結構示範與子 Tag 說明表
- **枚舉建表條件**：只為有明確可填值集合的欄位建表；枚舉值查找完整後全部列出

## 枚舉資料處理規則 {#enum-rules}

AI 生成文件時，所有枚舉資料必須遵守以下規則：

**查找來源優先順序**
1. Minecraft Wiki（[minecraft.wiki](https://minecraft.wiki/)）
2. 官方 Java Edition 資料包原始資料
3. 使用者提供的資料（若有）

**完整性要求**
- 枚舉值必須查找完整後再輸出，不得以「... 等」或「（其餘略）」代替
- 音效、粒子、物品、生物、效果等超長枚舉照列所有條目，無數量上限
- 每個枚舉條目必須填入分類欄與說明欄，不得留空

**無法確認的枚舉值處理**
- 若某個枚舉值無法確認說明，說明欄填 `—`，不得捏造
- 若整個枚舉列表無法查找完整，明確告知使用者並列出已確認的部分，標注「待補充」

**跨文件引用規則**
- 目標選擇器語法：只在 `target-selector.md` 中展開，其他文件一律用連結引用，不重複列出
- 座標系統說明：只在 `coordinate-system.md` 中展開，其他文件一律用連結引用
- 其他在獨立說明文件中已完整列出的內容，同樣改用連結引用

## 禁止行為速查 {#forbidden}

| 禁止行為 | 正確處理 |
|---------|---------|
| 保留任何佔位符文字 | 全部替換為實際內容 |
| 枚舉值寫「... 等」省略 | 查找完整後全部照列 |
| 欄位說明只寫資料類型 | 說明欄填實際用途描述 |
| 多個子指令欄位合併同一張表 | 每個子指令獨立 H3 + 語法 + 欄位表 |
| 選填欄位打 `✗` | 選填欄位留空 |
| 自行新增模板未定義的區塊 | 不得新增 |
| 條件依賴區塊無內容仍保留 | 無相依關係時整段含標題刪除 |
| 外部連結填實際 URL | 統一填 `#` |
| 目標選擇器語法在非專屬文件展開 | 只填連結引用 |
| 目錄錨點與實際標題不對應 | 錨點必須對應實際 H2/H3 標題文字 |
| 捏造無法確認的枚舉說明 | 說明欄填 `—`，並告知使用者 |
