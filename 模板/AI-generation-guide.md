# AI 生成說明（AI Generation Guide）

> 本文件說明如何讓 AI 依照對應模板，無誤差生成 Minecraft 指令說明文件

---

## 模板對應索引

| 使用場景 | 對應模板 |
|---------|---------|
| 單一指令說明（`/fill`、`/execute` 等）| `template-command-page.md` |
| 橫跨系統的獨立說明（目標選擇器、座標系統等）| `template-reference-doc.md` |
| 資料包總覽文件 | `template-datapack-illustrate.md` |
| 資料包單一元素說明（advancements、recipes 等）| `template-datapack-element.md` |
| NBT 總覽索引文件 | `template-nbt-overview.md` |
| NBT 大類說明（實體 / 物品 / 方塊實體）| `template-nbt-category.md` |

## 生成前自我核對 Checklist

生成任何文件前，AI 必須逐項確認：

- [ ] 已確認使用正確的模板
- [ ] 所有佔位符已替換（無任何「一句話說明」、`<值>`、`[大類名稱]` 殘留）
- [ ] 模板內所有 HTML 註解 `<!-- -->` 已全部刪除
- [ ] 區塊順序與模板一致，未自行新增或合併區塊
- [ ] 枚舉值已全部列出，無「... 等」省略
- [ ] 有版本限定的欄位或行為已標注 `[1.XX+]`
- [ ] 「條件依賴」區塊：有相依關係則填寫，無則整段刪除
- [ ] 外部連結區的 Wiki 連結必填，其餘選填，連結目的地統一填 `#`
- [ ] 最後更新日期已填入實際生成日期，格式 `YYYY-MM-DD`

## 標準生成 Prompt 格式

```
請依照 [模板名稱] 模板，生成以下內容的說明文件。

主題：[指令名稱 / 元素名稱 / NBT 大類名稱]
適用版本：[JE 1.XX+]
權限等級：[僅 command-page 需填]
子指令列表：[僅 command-page 需填，列出所有子指令名稱]
欄位定義來源：[貼上原始欄位資料，或指定參考來源]
枚舉值來源：[貼上完整枚舉列表]
版本限定欄位：[列出需標注 [1.XX+] 的欄位或行為]
條件依賴關係：[描述欄位間的相依或互斥規則，無則填「無」]

嚴格規則：
1. 模板區塊順序不得更動
2. 所有佔位符必須替換，HTML 註解全部刪除
3. 語言：中文主體，技術術語、ID、路徑、欄位名稱保留英文
4. 枚舉值全部列出，不得省略
5. 輸出純 Markdown，不加任何額外說明文字
```

## 各模板專屬規則

### `template-command-page.md`

- **語法區**：每種語法形式獨立一行，最小單元語法，不寫組合展開
- **子指令區**：每個子指令一個 `### 子指令名稱` H3，H3 下方先放該子指令的最小語法 code block，再接欄位說明表
- **無子指令的指令**（如 `/say`）：`子指令說明` 區塊整段替換為單一 `## 參數說明` 區塊
- **欄位說明表各欄定義：**
  - `欄位`：參數原始名稱，保留 `<>` / `[]` 括號
  - `類型`：`entity` / `int` / `string` / `enum` / `boolean` / `block` / `vec3` 等
  - `可填值 / 範圍`：枚舉值以 `/` 分隔；數值範圍用 `..`；無限制填 `—`
  - `預設`：選填參數的預設值；必填參數填 `—`
  - `說明`：實際用途描述，不得只寫類型名稱
- **超長枚舉（音效、粒子、物品等）**：在對應子指令 H3 下方新增 `#### 枚舉值` H4，加入分類欄，照列所有條目，不省略

---

### `template-reference-doc.md`

- **章節數量不固定**，依內容性質決定，章節標題替換佔位符
- **核心結構區**若無明確語法格式，整段替換為條列規則
- **枚舉型表格**加入分類欄；純說明型表格維持兩欄即可

---

### `template-datapack-illustrate.md`

- **命名空間資料夾表格**中的超連結目的地統一填 `#`
- **`pack_format` 表格**保持完整，不依版本截斷

---

### `template-datapack-element.md`

- **JSON 結構區**：只放精簡完整結構範本，選填欄位以實際預設值填入，不放所有可能欄位組合
- **欄位說明表**：`必填` 欄 `✓`，選填留空，不填 `✗`
- **枚舉值速查區**：只為有明確可填值集合的欄位建表；自由字串欄位（路徑、名稱）不建表
- **子欄位巢狀超過兩層**：每層獨立一個 `###` 子區塊說明，不在同一張表混列

---

### `template-nbt-overview.md`

- **大類索引表**連結目的地填 `#`
- **資料類型表與路徑語法表**保持完整，不截斷

---

### `template-nbt-category.md`

- **通用欄位區塊**只列所有子類型共通的 Tag，不列任何子類型專屬欄位
- **子類區塊數量**依實際子分類數量增減，每個子類一個 `##` 區塊
- **`compound[]` 欄位**：必須附上 SNBT 結構示範與子 Tag 說明表
- **枚舉建表條件**：只為有明確可填值集合的欄位建表

## 禁止行為速查（生成時強制遵守）

| 禁止行為 | 正確處理 |
|---------|---------|
| 保留任何佔位符文字 | 全部替換為實際內容 |
| 保留 `<!-- -->` HTML 註解 | 全部刪除 |
| 枚舉值寫「... 等」省略 | 全部照列，加分類欄 |
| 欄位說明只寫資料類型 | 說明欄填實際用途描述 |
| 多個子指令欄位合併同一張表 | 每個子指令獨立 H3 + 語法 + 欄位表 |
| 選填欄位打 `✗` | 選填欄位留空 |
| 自行新增模板未定義的區塊 | 不得新增 |
| 條件依賴區塊無內容仍保留 | 無相依關係時整段刪除 |
| 外部連結填實際 URL | 統一填 `#` |
