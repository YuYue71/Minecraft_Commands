## 模板對應索引 {#template-index}

| 使用場景 | 對應模板 |
|---------|---------|
| 單一指令說明（`/fill`、`/execute` 等）| `template-command-page.md` |
| 橫跨系統的獨立說明（目標選擇器、座標系統等）| `template-reference-doc.md` |
| 資料包總覽文件 | `template-datapack-illustrate.md` |
| 資料包**容器型**元素（`worldgen/`、`tags/` 等內含多個子分類資料夾）| `template-datapack-container.md` |
| 資料包**單一 JSON 型**元素（`advancements`、`recipes` 等，每個元素對應一個 `.json`）| `template-datapack-element.md` |
| NBT 總覽索引文件 | `template-nbt-overview.md` |
| NBT 大類說明（實體 / 物品 / 方塊實體）| `template-nbt-category.md` |

### 容器型 vs 單一 JSON 型 識別規則

生成資料包元素說明前，先判斷類型：

| 判斷問題 | 容器型 | 單一 JSON 型 |
|---------|--------|------------|
| 此元素本身是否為一個資料夾，內含多個**子分類資料夾**？ | ✓ 是 | — |
| 每個子分類資料夾是否各自有**不同的 JSON 結構**？ | ✓ 是 | — |
| 此元素對應的是**單一 `.json` 檔案**的格式定義？ | — | ✓ 是 |

常見容器型：`worldgen/`、`tags/`
常見單一 JSON 型：`advancements/`、`recipes/`、`loot_tables/`、`predicates/`、`damage_type/`

## 生成前強制詢問流程 {#pre-gen-inquiry}

收到生成請求後，**禁止直接輸出文件**。必須先執行以下詢問流程，等待使用者逐一確認後才能開始生成。

### 步驟一：確認基本資訊

詢問以下項目，所有欄位均需使用者明確回答，不得自行假設：

| 項目 | 詢問內容 | 缺少時的影響 |
|------|---------|------------|
| 主題名稱 | 指令 / 元素 / 大類的正確名稱 | 無法選擇模板 |
| 模板類型 | 確認是容器型還是單一 JSON 型（資料包）| 選錯模板導致結構錯誤 |
| 子指令列表 | （僅 command-page）所有子指令名稱 | 無法建立目錄與 H3 |
| 指令方塊支援 | （僅 command-page）`True` / `False` | Header 資訊缺失 |
| 條件依賴關係 | 欄位間是否有相依或互斥規則 | 條件依賴區塊無法判斷是否保留 |
| 共用內容 | 哪些欄位應改用連結引用（如目標選擇器）| 可能重複展開已有獨立文件的內容 |
| 是否有回傳值差異 | （僅 command-page）成功/失敗回傳值是否不同 | 回傳值區塊無法判斷是否保留 |

### 步驟二：確認枚舉與分段需求

- 此文件是否有超長枚舉（音效、粒子、物品、生物等）？
  - **有** → AI 自行查找，查找完成後先報告條目數量與分類，請使用者確認後再繼續
  - **無** → 繼續
- 預估文件長度是否會超過單次輸出限制？
  - **會** → 告知使用者將採用分段生成流程（見「長文件分段生成規則」）
  - **不會** → 直接進入步驟三

### 步驟三：取得確認後才生成

整理摘要請使用者確認，格式如下：

```
【生成前確認摘要】
模板：template-XXX.md
主題：XXX
適用版本：JE ≤ 1.20.4
（其餘收集到的資訊條列）

確認無誤請回覆「確認」，需修改請直接說明。
```

**收到「確認」後才輸出文件。**

## 生成前核對 Checklist {#checklist}

生成任何文件前，逐項確認後再輸出：

- [ ] 使用正確的模板，容器型 / 單一 JSON 型已正確識別
- [ ] 所有佔位符已替換，無任何「一句話說明」、`<值>`、`[大類名稱]` 殘留
- [ ] `> 選填：...` 標注若無內容，整行刪除
- [ ] 「條件依賴」區塊無內容時，整段含標題一起刪除
- [ ] 「回傳值」區塊無差異時，整段含標題一起刪除
- [ ] 目錄索引的連結文字使用**中文**，連結目的地統一填 `#`，不填實際錨點
- [ ] 目錄索引無重複條目，每個區塊只出現一次
- [ ] 所有 H2/H3 標題後不附加 `{#anchor-id}` 語法，錨點標注全部刪除
- [ ] 所有站內錨點使用 `(#anchor-id)` 格式，不帶 `https://` 或任何域名
- [ ] 枚舉值已查找完整資料並全部列出，無「... 等」省略
- [ ] 已棄用的枚舉值標注「已棄用 Deprecated」
- [ ] 版本基準統一為 `JE ≤ 1.20.4`，Header 無其他版本格式
- [ ] 外部連結 Wiki 項目必填，連結目的地統一填 `#`
- [ ] 最後更新日期填入實際生成日期，格式 `YYYY-MM-DD`
- [ ] 輸出純 Markdown，無額外說明文字包裹

## 標準生成 Prompt 格式 {#prompt-format}

```
請依照 [模板名稱] 模板，生成以下內容的說明文件。

主題：[指令名稱 / 元素名稱 / NBT 大類名稱]
模板類型：[容器型 / 單一 JSON 型 / 指令 / 其他]
適用版本：JE ≤ 1.20.4
權限等級：[僅 command-page 需填]
指令方塊：[僅 command-page 需填，True / False]
子指令列表：[僅 command-page 需填]
回傳值差異：[僅 command-page 需填，有 / 無]
條件依賴關係：[描述欄位間相依或互斥規則，無則填「無」]
共用內容引用：[列出應改用連結引用的內容，如目標選擇器、座標系統]

枚舉資料：由 AI 自行查找，規則見「枚舉資料處理規則」
分段生成：[需要 / 不需要]

嚴格規則：
1. 模板區塊順序不得更動
2. 所有佔位符必須替換，無殘留
3. 語言：中文主體，技術術語、ID、路徑、欄位名稱保留英文
4. 枚舉值查找完整後全部列出，不省略
5. 輸出純 Markdown，不加任何額外說明文字
```

## 各模板專屬規則 {#template-rules}

### `template-command-page.md`

- **Header**：`指令方塊` 欄位必填 `True` / `False`；版本統一 `JE ≤ 1.20.4`
- **語法區**：每種語法形式獨立一行，只寫最小單元語法，不展開組合
- **子指令區**：每個子指令一個 `###` H3，H3 下方先放最小語法 code block，再接欄位說明表
- **無子指令的指令**（如 `/say`）：`子指令說明` 區塊整段替換為單一 `## 參數說明` 區塊
- **欄位說明表各欄定義：**
  - `欄位`：參數原始名稱，保留 `<>` / `[]` 括號
  - `類型`：`entity` / `int` / `string` / `enum` / `boolean` / `block` / `vec3` / `float` 等
  - `可填值 / 範圍`：枚舉值以 `/` 分隔；數值範圍用 `..`；無限制填 `—`
  - `預設`：選填參數的預設值；必填填 `—`
  - `說明`：實際用途描述，不得只寫類型；共用語法改用連結引用
- **超長枚舉**（音效、粒子、物品等）：在子指令 H3 下方新增 `#### 枚舉值` H4，加入分類欄，全部照列
- **目標選擇器參數**：說明欄填「語法見 [目標選擇器](#)」，不在此展開
- **回傳值區塊**：有成功/失敗差異時填寫，無差異時整段刪除

---

### `template-datapack-container.md`

- **識別條件**：此元素本身為資料夾，內含多個子分類資料夾，每個子分類各有不同 JSON 結構
- **子分類索引表**：列出所有子分類，連結目的地填對應 H2 錨點
- **每個子分類**獨立一個 `##` H2 區塊，包含：路徑說明 → JSON 結構 → 欄位說明 → 枚舉值速查（如有）
- **多 type 結構**：依 `type` 值不同而結構不同時，每種 type 各放一個 `#### type = "..."` 結構範本
- **子分類數量**依實際內容增減，同步更新目錄

---

### `template-datapack-element.md`

- **識別條件**：此元素對應單一 `.json` 檔案的格式定義
- **JSON 結構區**：
  - 單一結構：只放一個精簡完整範本
  - 多 type 結構：依 `type` 值不同，每種 type 各放一個 `#### type = "..."` 結構範本
- **欄位說明表**：必填打 `✓`，選填留空，不填 `✗`
- **枚舉值速查區**：只為有明確可填值集合的欄位建表；自由字串欄位不建表
- **子欄位巢狀超過兩層**：每層獨立一個 `###` 子區塊說明，不在同一張表混列

---

### `template-reference-doc.md`

- **章節數量不固定**，依內容性質決定，章節標題替換佔位符後同步更新目錄
- **核心結構區**若無明確語法格式，整段替換為條列規則
- **枚舉型表格**加入分類欄；純說明型表格維持兩欄

---

### `template-nbt-overview.md`

- **大類索引表**連結目的地填 `#`
- **資料類型表與路徑語法表**保持完整，不截斷

---

### `template-nbt-category.md`

- **通用欄位區塊**只列所有子類型共通的 Tag，不列任何子類型專屬欄位
- **子類區塊數量**依實際子分類數量增減，每個子類一個 `##` 區塊，同步更新目錄
- **`compound[]` 欄位**：必須附上 SNBT 結構示範與子 Tag 說明表
- **枚舉建表條件**：只為有明確可填值集合的欄位建表，全部列出

## 枚舉資料處理規則 {#enum-rules}

**查找來源優先順序**
1. Minecraft Wiki（`minecraft.wiki`）
2. 官方 Java Edition 資料包原始資料
3. 使用者提供的資料（若有）

**完整性要求**
- 枚舉值必須查找完整後再輸出，不得以「... 等」或「（其餘略）」代替
- 音效、粒子、物品、生物、效果等超長枚舉照列所有條目，無數量上限
- 每個枚舉條目必須填入分類欄與說明欄，不得留空
- 已棄用（Deprecated）的值仍須列出，說明欄標注「已棄用 Deprecated」

**無法確認的枚舉值處理**
- 若某個枚舉值無法確認說明，說明欄填 `—`，不得捏造
- 若整個枚舉列表無法查找完整，明確告知使用者並列出已確認的部分，標注「待補充」

**跨文件引用規則**
- 目標選擇器語法：只在 `target_selectors.md` 中展開，其他文件一律用連結引用
- 座標系統說明：只在 `coordinates.md` 中展開，其他文件一律用連結引用
- 其他在獨立說明文件中已完整列出的內容，同樣改用連結引用

## 長文件分段生成規則 {#long-doc-rules}

當預估文件內容超過單次輸出限制時（如音效指令 500+ 枚舉、worldgen/ 14 個子分類），採用以下分段流程：

### 分段流程

1. **第一段：主體骨架**
   - 生成 Header、目錄、概覽、所有區塊的標題與空白佔位，枚舉表格只寫表頭
   - 在各待補充位置標注：`<!-- TODO: 待補充 - [區塊說明] -->`
   - 輸出完成後詢問：「骨架已完成，請確認後告知要繼續生成哪個區塊」

2. **後續段：逐區塊填入**
   - 依使用者指定的區塊名稱，輸出該區塊的完整內容
   - 使用者將內容貼入對應的 `<!-- TODO -->` 位置
   - 完成後告知下一個待補充區塊，繼續詢問

3. **收尾確認**
   - 所有 `<!-- TODO -->` 填完後，提示使用者進行最終檢查

### TODO 標注格式

```markdown
<!-- TODO: 待補充 - 子分類A 枚舉值速查 -->
| 值 | 分類 | 說明 |
|----|------|------|

<!-- TODO: 待補充 - 子分類B JSON 結構與欄位說明 -->
```

## 錨點格式規則 {#anchor-format}

**目錄索引規則**
- 連結文字使用**中文**
- 連結目的地統一填 `#`，不填實際錨點 ID（由使用者自行修改）
- 每個區塊在目錄中只出現一次，不得重複

```markdown
<!-- 正確 -->
- [語法](#)
- [子指令說明](#)
  - [子指令A](#)

<!-- 禁止 -->
- [語法](https://example.com#syntax)
- [語法](https://www.google.com/search?q=%23syntax)
- [語法](#syntax)   ← 禁止填入實際錨點，統一填 #
```

**標題格式規則**
- H2/H3 標題後**不附加** `{#anchor-id}` 語法，全部刪除
- 標題只保留文字本身

```markdown
<!-- 正確 -->
## 語法 (Syntax)

<!-- 禁止 -->
## 語法 (Syntax) {#syntax}
```

## 禁止行為速查 {#forbidden}

| 禁止行為 | 正確處理 |
|---------|---------|
| 收到請求後未詢問直接生成 | 執行生成前強制詢問流程，取得確認後才輸出 |
| 容器型元素使用 single-element 模板 | 先識別類型，選用 `template-datapack-container.md` |
| 容器型元素的 JSON 結構區列出子分類名稱 | 子分類各自有獨立 H2 區塊，JSON 結構在各自區塊內 |
| 保留任何佔位符文字 | 全部替換為實際內容 |
| 枚舉值寫「... 等」省略 | 查找完整後全部照列 |
| 已棄用值不列出 | 仍須列出，說明欄標注「已棄用 Deprecated」 |
| 欄位說明只寫資料類型 | 說明欄填實際用途描述 |
| 多個子指令欄位合併同一張表 | 每個子指令獨立 H3 + 語法 + 欄位表 |
| 選填欄位打 `✗` | 選填欄位留空 |
| 自行新增模板未定義的區塊 | 不得新增 |
| 條件依賴 / 回傳值區塊無內容仍保留 | 無內容時整段含標題刪除 |
| 外部連結填實際 URL | 統一填 `#` |
| 錨點連結帶入 `https://` 或任何域名 | 站內錨點只用 `(#anchor-id)` 格式 |
| 目錄連結文字使用英文 | 目錄連結文字統一使用中文 |
| 目錄連結目的地填入實際錨點 ID | 統一填 `#`，由使用者自行修改 |
| H2/H3 標題後附加 `{#anchor-id}` | 標題後不附加任何錨點語法 |
| 目錄中同一區塊重複出現 | 每個區塊只列一次 |
| 目標選擇器 / 座標語法在非專屬文件展開 | 只填連結引用 |
| 捏造無法確認的枚舉說明 | 說明欄填 `—`，並告知使用者 |
| 長文件直接硬輸出超過限制的內容 | 採用分段生成流程，先輸出骨架再逐區塊填入 |
| Header 版本格式不統一 | 統一使用 `JE ≤ 1.20.4` |
