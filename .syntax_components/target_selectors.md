# 目標選擇器（Target Selectors）

## 核心五大基礎選擇器
> 用於在執行指令時, 指定要對哪些實體（Entity）或玩家進行操作的核心變數：

| 選擇器 | 英文全稱 | 預設選取目標 | 預設排序邏輯與記憶體特性 |
| :--- | :--- | :--- | :--- |
| `@p` | Nearest Player | 距離執行點最近的玩家 | 依空間三維距離由近到遠排序, 忽略已死亡的玩家 |
| `@r` | Random Player | 隨機單一在線玩家 | 呼叫偽隨機數生成器隨機選取一人, 適用於抽獎系統 |
| `@a` | All Players | 伺服器當前所有在線玩家 | 包含生存, 創造, 旁觀等所有模式的玩家, 預設不排序 |
| `@e` | All Entities | 當前已載入區塊內的所有實體 | 包含玩家, 生物, 掉落物, 箭矢與盔甲架, 全域搜尋負擔較重 |
| `@s` | Executor | 當前指令的執行發送者本體 | 精準指向發送指令的實體或方塊本身, 執行效能最高 |

## 常用過濾引數矩陣 (Arguments)
> 附加在選擇器後方的中括號 `[]` 內, 用於進一步篩選目標（多個引數以 ", " 隔開, 部分引數前方加上 `!` 代表「非 / 取反」）：

| 引數群組 | 引數名稱 | 接受數值型態 | 篩選功能與說明 |
| :--- | :--- | :--- | :--- |
| **空間限制** | `distance` | 數值區間（如 `..10`） | 限制目標與執行點之間的距離（如 `..5` 為 5 格內, `5..` 為 5 格外） |
| | `x`, `y`, `z` | 浮點數 | 重新定義選取範圍的中心點座標（預設為執行指令的位置） |
| | `dx`, `dy`, `dz` | 浮點數 | 定義一個由中心點向外延伸的三維立方體選取區域 |
| **數量與排序**| `limit` | 正整數 | 限制回傳的目標最大數量（舊版本為 `c`） |
| | `sort` | `nearest`, `furthest`, `random`, `arbitrary` | 改變選取目標的排序邏輯（如搭配 `limit=1` 可以選取最遠或隨機目標） |
| **實體屬性** | `type` | 實體識別碼（如 `zombie`） | 限制特定的生物或實體種類, 支援取反（如 `type=!player`） |
| | `gamemode` | 遊戲模式（如 `creative`） | 限制目標玩家的遊戲模式, 支援取反（如 `gamemode=!spectator`） |
| | `level` | 數值區間 | 篩選玩家的當前經驗值等級區間 |
| **變數與標籤**| `tag` | 自訂字串 | 篩選在記憶體中附加了特定標籤的實體, 支援取反（如 `tag=!vip`） |
| | `team` | 隊伍識別碼 | 篩選屬於特定計分板隊伍的成員, 支援取反（如 `team=!red`） |
| | `scores` | 嵌套鍵值對（如 `{kill=1..}`）| 篩選計分板分數符合特定區間的目標 |
