# 目標選擇器 (Target Selectors)

> **分類:** `目標選擇器` / `座標系統` | **適用版本:** `JE 1.20.4`

---

## 概覽 (Overview)

用於指令中精準篩選與指定指令執行對象或座標目標的核心系統.

---

## 核心結構 (Core Structure)

```
@<基礎選擇器>[<參數1>=<值1>,<參數2>=<值2>,...]

```

---

## 基礎選擇器 (Base Selectors)

| 值 | 分類 | 說明 |
| --- | --- | --- |
| `@p` | 玩家 | 選擇距離執行點最近的玩家 |
| `@a` | 玩家 | 選擇所有玩家 |
| `@r` | 玩家 | 隨機選擇一名玩家 |
| `@e` | 實體 | 選擇所有實體(包含玩家與非玩家生物,掉落物等) |
| `@s` | 實體 | 選擇當前指令的執行者本身 |

---

## 選擇器參數 (Selector Arguments)

### 空間與距離參數

| 項目 | 類型 / 值 | 預設 | 說明 |
| --- | --- | --- | --- |
| `x`,`y`,`z` | `double` | 執行點座標 | 定義搜尋基準點的絕對座標 |
| `distance` | `range` | `0..` | 篩選與搜尋基準點距離在特定範圍內的實體 |
| `dx`,`dy`,`dz` | `double` | - | 篩選位於以搜尋基準點為起點之包圍盒(Bounding Box)內的實體 |

### 屬性與過濾參數

| 項目 | 類型 / 值 | 預設 | 說明 |
| --- | --- | --- | --- |
| `type` | `entity_id` | - | 篩選特定實體類型,支援 `!` 反向排除(如 `type=!minecraft:player`) |
| `tag` | `string` | - | 篩選擁有特定標籤的實體,支援 `!` 反向排除 |
| `team` | `string` | - | 篩選屬於特定隊伍的實體,支援 `!` 反向排除或空值檢測 |
| `name` | `string` | - | 篩選特定名稱的實體,支援 `!` 反向排除 |
| `nbt` | `snbt` | - | 篩選符合特定 NBT 資料結構的實體,支援 `!` 反向排除 |
| `scores` | `object` | - | 篩選特定計分板分數範圍的實體(格式:`{obj=1..10}`) |
| `predicate` | `string` | - | 篩選符合特定資料包謂詞(Predicate)條件的實體,支援 `!` 反向排除 |

### 玩家專屬參數

| 項目 | 類型 / 值 | 預設 | 說明 |
| --- | --- | --- | --- |
| `level` | `range` | `0..` | 篩選經驗等級在特定範圍內的玩家 |
| `gamemode` | `enum` | - | 篩選處於特定遊戲模式的玩家,支援 `!` 反向排除 |
| `advancements` | `object` | - | 篩選特定成就完成狀態的玩家 |

### 視角與旋轉參數

| 項目 | 類型 / 值 | 預設 | 說明 |
| --- | --- | --- | --- |
| `x_rotation` | `range` | `-90..90` | 篩選垂直視角(俯仰角)在特定範圍內的實體 |
| `y_rotation` | `range` | `-180..180` | 篩選水平視角(偏航角)在特定範圍內的實體 |

### 排序與限制參數

| 項目 | 類型 / 值 | 預設 | 說明 |
| --- | --- | --- | --- |
| `limit` | `int` | 見下方條件依賴 | 限制選擇器回傳的最大實體數量 |
| `sort` | `enum` | 見下表 | 定義多個實體符合條件時的傳回排序方式 |

#### `sort` 枚舉值

| 值 | 分類 | 說明 |
| --- | --- | --- |
| `nearest` | 距離排序 | 由近到遠排序 |
| `furthest` | 距離排序 | 由遠到近排序 |
| `random` | 隨機排序 | 隨機打亂順序 |
| `arbitrary` | 效能優化 | 不排序,直接回傳最先找到的實體(效能最佳) |

---

## 條件依賴 (Conditional Fields)

| 條件 | 生效欄位 | 失效 / 忽略欄位 |
| --- | --- | --- |
| 使用 `@p` 時 | `limit=1` , `sort=nearest` 成為預設值 | - |
| 使用 `@a` 時 | `sort=arbitrary` 成為預設值 | 無法使用 `type` 選擇非玩家實體 |
| 使用 `@r` 時 | `limit=1` , `sort=random` 成為預設值 | - |
| 使用 `@e` 時 | `sort=arbitrary` 成為預設值 | - |

---

## 注意事項 (Notes)

* 同一選擇器中可重複使用 `tag`, `team`, `scores`, `name`, `type`, `predicate`, `nbt` 參數以實現 AND 邏輯交集.
* `x`,`y`,`z` 參數若未搭配 `distance` 或 `dx`,`dy`,`dz` 使用,僅會改變搜尋中心點,不會限制範圍.
* 反向排除條件(如 `type=!player`)不支援重複寫入相同的排除值.
